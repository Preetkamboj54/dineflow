# Plan 09-02: Customer Pages Migration

## Objective

Migrate all 8 customer-facing pages to use Tailwind utility classes instead of any remaining custom CSS classes. Keep the semantic `card`, `btn-primary`, `btn-secondary`, `btn-sm`, `btn-danger`, `restaurants-grid`, `restaurant-card`, `menu-item`, `form-group` component classes (which stay in `@layer components`). Replace anything else that was previously defined in `style.css` as a utility duplicate.

---

## Class Replacement Reference

These mappings apply across all tasks in this plan:

| Old class (from style.css) | Tailwind replacement |
|---|---|
| `text-muted` | `text-[var(--text-muted)]` |
| `text-primary` | `text-[var(--primary)]` |
| `text-secondary` | `text-[var(--secondary)]` |
| `text-danger` | `text-[var(--danger)]` |
| `container` (max-width layout) | `max-w-[1200px] mx-auto my-8 px-6` |
| `hero-section` | `text-center py-12` |
| `shadow` | `shadow-md` |
| `shadow-lg` | `shadow-lg` |
| `rounded-lg` | `rounded-xl` |
| `space-y-4 > * + *` | use `space-y-4` (Tailwind has this) |
| `line-clamp-2` | `line-clamp-2` (Tailwind has this since v3.3) |

Keep: `card`, `restaurants-grid`, `restaurant-card`, `menu-item`, `form-group`, `btn-*`, `stats-card`, `kanban-*`
