  <files>
    server/models/Order.js,

    server/controllers/orderController.js, 
    server/routes/orderRoutes.js, 
    server/server.js
  </files>
  <behavior>
    - Checkout: Validate that all item IDs exist, re-calculate total price on server using current DB prices (do not trust client-sent totals), and create Order + OrderItems.
    - History: Retrieve all orders for the authenticated user, sorted by date.
    - Status: Orders should start as 'Pending'.
  </behavior>
  <action>
    1. Create `Order` model: userId (ref), restaurantId (ref), items (ref OrderItem), totalAmount, paymentMethod (COD/Pay at Restaurant), status (Pending, Preparing, Ready, Completed, Cancelled), createdAt.
    2. Create `OrderItem` model: orderId (ref), menuItemId (ref), quantity, priceAtTimeOfOrder.
    3. Implement `orderController` with:
       - `createOrder`: Validates items, calculates total, saves order.
       - `getUserOrders`: Fetches orders for `req.user.id`.
       - `getOrderById`: Fetches details of a specific order.
    4. Wire up `orderRoutes` (all protected by `verifyToken`) and register in `server.js` under `/api/orders`.
  </action>
  <verify>
    <automated>
      curl -X POST http://localhost:5000/api/orders -H "Authorization: Bearer ..." -d '{"restaurantId": "...", "items": [{"menuItemId": "...", "quantity": 1}]}' returns 201
    </automated>
  </verify>
  <done>Order backend is implemented with server-side price validation and order history endpoints.</done>
</task>
